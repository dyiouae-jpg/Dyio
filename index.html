DYIO Admin Setup

Connect your Supabase project to enable cloud sync and multi-user access. This only takes 2 minutes.

1

Create a free Supabase project

Go to supabase.com → New Project. Choose any name and a strong database password. Wait ~2 minutes for it to spin up.

2

Run the SQL setup

In Supabase → SQL Editor, paste and run this:

create table delegates (
  id uuid primary key default gen_random_uuid(),
  first_name text, last_name text, email text, phone text,
  school text, grade text, nationality text,
  participation_type text default 'individual',
  team_name text, team_size text, pillar text,
  competitions text[] default '{}',
  status text default 'pending',
  payment_status text default 'unpaid',
  notes text default '',
  added_by text,
  registered_at timestamptz default now()
);
create table activity_log (
  id uuid primary key default gen_random_uuid(),
  text text, user_email text,
  created_at timestamptz default now()
);
alter table delegates enable row level security;
alter table activity_log enable row level security;
create policy "auth_all" on delegates for all to authenticated using (true) with check (true);
create policy "auth_all" on activity_log for all to authenticated using (true) with check (true);
3

Copy your project credentials

Go to Settings → API in Supabase. Copy your Project URL and anon public key.

Please enter both your Supabase URL and API key.

These credentials are stored in your browser. You'll only need to enter them once per device.

🏷️ Discount Codes

⚠️ For codes your registration website can validate, run the one-time SQL below (Supabase → SQL Editor). Until then codes are stored portal-only.
One-time SQL + website hookup snippet

1) Supabase → SQL Editor, run once:

create table if not exists discount_codes (
  id uuid primary key default gen_random_uuid(),
  code text unique not null, kind text default 'percent',
  value numeric default 10, active boolean default true,
  expires date, max_uses int, used int default 0,
  created_at timestamptz default now());
alter table discount_codes enable row level security;
create policy "public_read_active" on discount_codes for select to anon using (active = true);
create policy "auth_all_codes" on discount_codes for all to authenticated using (true) with check (true);

2) On your registration website (same Supabase URL + anon key), validate a code like this:

async function checkDiscount(code){
  const { data } = await sbClient.from('discount_codes')
    .select('*').eq('code', code.toUpperCase()).maybeSingle();
  if(!data) return null;
  if(data.expires && new Date(data.expires) < new Date()) return null;
  if(data.max_uses && data.used >= data.max_uses) return null;
  return data; // {kind:'percent'|'fixed', value}
}

Because the website reads the same Supabase, any code you create or deactivate here goes live on the site instantly — no website redeploy needed.

📸 Connect Instagram (real analytics)

Pulls live follower count, post performance and engagement rate straight from Instagram. Needs an Instagram Business or Creator account linked to a Facebook Page (free, 2 min in Instagram settings).

How to get the token (free, 3 minutes)
  1. Make sure your Instagram is a Business/Creator account linked to a Facebook Page (Instagram app → Settings → Business tools).
  2. Go to developers.facebook.com/tools/explorer (log in with the Facebook account that owns the Page).
  3. Under Permissions add: instagram_basic, pages_show_list, instagram_manage_insights → click Generate Access Token.
  4. Copy the token and paste it above. Tokens expire after ~1-2 hours (or generate a long-lived 60-day one via the token debugger) — just re-paste when needed.

✍️ Or log a snapshot manually (works for any platform)
